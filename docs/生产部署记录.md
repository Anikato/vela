## 一、部署架构

```
┌──────────────────────┐    ┌──────────────────┐    ┌─────────────────────────────┐
│ OpenResty (1Panel)   │───▶│ Vela (Next.js 16)│───▶│ PostgreSQL 18               │
│ :80 / :443           │    │ :3000 (仅本地)    │    │ 1Panel-postgresql-vxdz      │
│ 反向代理 + HTTPS      │    │ Docker 容器       │    │ :5432 (已有实例，复用)        │
└──────────────────────┘    └──────────────────┘    └─────────────────────────────┘
                                   │
                                   ▼
                            /data/vela/data/uploads/
                            (绑定挂载到项目目录，随项目迁移)
```

**关键设计**：

- **复用已有 PostgreSQL**：不单独起 PG 容器，节省内存（~200 MB），通过 Docker 网络 `1panel-network` 连接
- **文件存储在项目目录**：上传文件保存在 `/data/vela/data/uploads/`，绑定挂载到容器内，迁移时直接打包整个 `/data/vela` 目录即可
- **仅监听 127.0.0.1:3000**：不暴露到公网，通过 OpenResty 反向代理提供 HTTPS 访问

## 五、日常运维命令

### 查看日志

```bash
# 实时日志
docker logs vela-app -f

# 最近 100 行
docker logs vela-app --tail 100
```

### 重启

```bash
cd /data/vela
docker compose -f docker-compose.prod.yml restart
```

### 更新部署

```bash
cd /data/vela

# 1. 拉取最新代码
git pull

# 2. 重新构建并启动（自动滚动更新）
docker compose -f docker-compose.prod.yml up -d --build

# 3. 如果有数据库 Schema 变更

  -v "$(pwd)":/app -w /app \
  -e DATABASE_URL="postgresql://vela:VelaP@ss2026!@127.0.0.1:5432/vela" \
  node:22-alpine \
  sh -c "corepack enable pnpm && pnpm install --frozen-lockfile && npx drizzle-kit push --force"
```

```bash
# 进入 psql
docker exec -it 1Panel-postgresql-vxdz psql -U vela -d vela

# 完整备份
docker exec 1Panel-postgresql-vxdz pg_dump -U vela vela > /data/vela/data/backup-$(date +%Y%m%d).sql

# 恢复备份
docker exec -i 1Panel-postgresql-vxdz psql -U vela vela < /data/vela/data/backup-20260308.sql
```

```bash

0 2 * * * docker exec 1Panel-postgresql-vxdz pg_dump -U vela vela | gzip > /data/vela/data/backup-$(date +\%Y\%m\%d).sql.gz
```

## 六、迁移指南

### 迁移到新服务器

本项目设计为**一个目录即全部**，迁移步骤：

#### 1. 在旧服务器上备份

```bash
# 备份数据库
docker exec 1Panel-postgresql-vxdz pg_dump -U vela vela > /data/vela/data/backup-migrate.sql

# 打包整个项目目录（含上传文件 + .env）
cd /data
tar czf vela-full-backup.tar.gz vela/
```

#### 2. 传输到新服务器

```bash
scp /data/vela-full-backup.tar.gz user@new-server:/data/
```

#### 3. 在新服务器上恢复

```bash
# 解压
cd /data
tar xzf vela-full-backup.tar.gz

# 确保 Docker 已安装
docker --version
docker compose version

# 如果新服务器也有 1Panel + PostgreSQL：
#   1. 创建 vela 用户和数据库（见下方）
#   2. 恢复数据
#   3. 启动应用

# 创建数据库用户
docker exec <pg-container> psql -U <admin-user> -c "CREATE USER vela WITH PASSWORD 'VelaP@ss2026!';"
docker exec <pg-container> psql -U <admin-user> -c "CREATE DATABASE vela OWNER vela;"

# 恢复数据
docker exec -i <pg-container> psql -U vela vela < /data/vela/data/backup-migrate.sql

# 修改 .env 中的 DATABASE_URL（如果 PG 容器名不同）
vi /data/vela/.env

# 修改 docker-compose.prod.yml 中的构建参数 DATABASE_URL
vi /data/vela/docker-compose.prod.yml

# 确保上传目录权限正确
chown -R 1001:1001 /data/vela/data/uploads

# 确保 Docker 网络存在
docker network create 1panel-network 2>/dev/null || true

# 构建并启动
cd /data/vela
docker compose -f docker-compose.prod.yml up -d --build
```

#### 4. 如果新服务器没有现成的 PostgreSQL

修改 `docker-compose.prod.yml`，添加独立的 PG 服务：

```yaml
services:
  app:
    # ... 现有配置不变，但修改：
    depends_on:
      db:
        condition: service_healthy

  db:
    image: postgres:18-alpine
    container_name: vela-db
    restart: unless-stopped
    environment:
      POSTGRES_DB: vela
      POSTGRES_USER: vela
      POSTGRES_PASSWORD: VelaP@ss2026!
    volumes:
      - ./data/pgdata:/var/lib/postgresql/data
    healthcheck:
      test: ['CMD-SHELL', 'pg_isready -U vela']
      interval: 5s
      timeout: 5s
      retries: 5
```

同时将 `.env` 中的 `DATABASE_URL` 改为：

```
DATABASE_URL=postgresql://vela:VelaP@ss2026!@db:5432/vela
```

## 七、反向代理配置

### OpenResty / Nginx 配置模板

```nginx
upstream vela {
    server 127.0.0.1:3000;
    keepalive 10;
}

server {
    listen 443 ssl http2;
    listen [::]:443 ssl http2;
    server_name your-domain.com;

    ssl_certificate     /path/to/fullchain.pem;
    ssl_certificate_key /path/to/privkey.pem;
    ssl_protocols TLSv1.2 TLSv1.3;

    client_max_body_size 50m;

    # 静态资源长期缓存
    location /_next/static {
        proxy_pass http://vela;
        proxy_cache_valid 200 365d;
        add_header Cache-Control "public, immutable";
    }

    # 上传文件缓存
    location /uploads {
        proxy_pass http://vela;
        proxy_cache_valid 200 30d;
        add_header Cache-Control "public, max-age=2592000";
    }

    # 应用代理
    location / {
        proxy_pass http://vela;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

# HTTP 重定向到 HTTPS
server {
    listen 80;
    server_name your-domain.com;
    return 301 https://$host$request_uri;
}
```

```
    reverse_proxy localhost:3000
```

## 八、文件存储说明

### 当前方案：本地存储（项目目录绑定挂载）

上传的文件存储在 **`/data/vela/data/uploads/`** 目录下，通过 Docker 绑定挂载映射到容器内的 `/app/public/uploads/`。

**优点**：
- 文件直接在项目目录中，`ls` 即可查看
- 迁移时随项目一起打包，无需单独导出 Docker Volume
- 备份简单：`tar czf uploads-backup.tar.gz data/uploads/`

**注意**：
- 容器内进程以 uid 1001 (nextjs) 运行，目录权限需要 `chown -R 1001:1001 data/uploads/`
- `.gitignore` 和 `.dockerignore` 已配置忽略 `data/` 目录

### 切换到 S3/R2（推荐大规模生产）

修改 `.env`：

```bash
STORAGE_TYPE=s3
S3_ENDPOINT=https://xxx.r2.cloudflarestorage.com
S3_REGION=auto
S3_BUCKET=vela-media
S3_ACCESS_KEY=your-access-key
S3_SECRET_KEY=your-secret-key
S3_PUBLIC_URL=https://media.your-domain.com
```

然后重新构建：`docker compose -f docker-compose.prod.yml up -d --build`

## 九、排障指南

### 容器无法启动

```bash
# 查看详细错误
docker logs vela-app

# 常见原因：
# 1. 数据库连接失败 → 检查 PG 容器是否运行、网络是否正确
# 2. 端口被占用 → ss -tlnp | grep 3000
# 3. 权限问题 → chown -R 1001:1001 data/uploads/
```

### 构建失败 (prerender error)

构建时需要连接数据库预渲染页面。确保：
1. PostgreSQL 容器正在运行
2. `docker-compose.prod.yml` 中 `build.args.DATABASE_URL` 使用 `127.0.0.1`（构建时用 `network: host`）
3. `build.network: host` 配置存在

### 上传文件丢失

```bash
# 检查挂载是否正确
docker inspect vela-app --format '{{range .Mounts}}{{.Type}} {{.Source}} → {{.Destination}}{{"\n"}}{{end}}'

# 应显示：bind /data/vela/data/uploads → /app/public/uploads

# 检查权限
ls -la /data/vela/data/uploads/
# uploads 目录应属于 1001:1001
```

### 数据库连接超时

```bash
# 检查 PG 容器
docker ps --filter name=1Panel-postgresql

# 检查网络连通性
docker exec vela-app sh -c "wget -q -O- http://1Panel-postgresql-vxdz:5432 2>&1 || echo 'Port reachable (expected error is normal)'"

# 确保在同一 Docker 网络
docker network inspect 1panel-network --format '{{range .Containers}}{{.Name}} {{end}}'
```

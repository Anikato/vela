## 快速部署（Docker Compose）

### 1. 准备环境变量

在项目根目录创建 `.env` 文件：

```bash
# 必填
DB_PASSWORD=your-strong-db-password
NEXTAUTH_SECRET=$(openssl rand -base64 32)
SITE_URL=https://your-domain.com

# 文件存储（二选一）
STORAGE_TYPE=local          # 简单部署用本地存储
# STORAGE_TYPE=s3           # 生产推荐用 S3/R2

# S3 存储配置（STORAGE_TYPE=s3 时必填）
# S3_ENDPOINT=https://xxx.r2.cloudflarestorage.com
# S3_REGION=auto
# S3_BUCKET=vela-media
# S3_ACCESS_KEY=your-access-key
# S3_SECRET_KEY=your-secret-key
# S3_PUBLIC_URL=https://media.your-domain.com

# SMTP 密码加密密钥（强烈推荐）
SMTP_ENCRYPTION_KEY=$(openssl rand -hex 32)
```

### 2. 构建并启动

```bash
docker compose up -d --build
```

### 3. 初始化数据库

```bash
# 推送表结构
docker compose exec app npx drizzle-kit push

# 运行种子数据（创建默认管理员账号）
docker compose exec app npx tsx scripts/seed.ts
```

### 4. 访问

- 前台：`http://localhost:3000`
- 后台：`http://localhost:3000/admin/login`
- 默认账号：见种子脚本输出

## 文件存储方案

### 方案一：本地存储（简单部署）

文件存储在容器内 `/app/public/uploads/`，通过 Docker volume 持久化。

```yaml
# docker-compose.yml 已配置
volumes:
  - uploads:/app/public/uploads
```

**特点**：
- 零配置，开箱即用
- 文件通过 Next.js 直接提供 HTTP 服务（`/uploads/xxx`）
- 容器重建后数据不丢失（volume 持久化）

**限制**：
- 不支持多实例水平扩展（文件不跨节点共享）
- 大量文件时 Next.js 服务静态资源性能一般
- 备份需要单独导出 volume

**备份本地文件**：
```bash
docker compose cp app:/app/public/uploads ./backup-uploads
```

### 方案二：S3/R2 对象存储（生产推荐）

支持 AWS S3、Cloudflare R2、MinIO 等 S3 兼容服务。

```bash
STORAGE_TYPE=s3
S3_ENDPOINT=https://xxx.r2.cloudflarestorage.com
S3_REGION=auto
S3_BUCKET=vela-media
S3_ACCESS_KEY=your-key
S3_SECRET_KEY=your-secret
S3_PUBLIC_URL=https://media.your-domain.com  # CDN 域名
```

**特点**：
- 文件与应用分离，支持多实例部署
- 天然 CDN 加速（R2 免出站流量费）
- 容器无状态，可随时重建
- 自动按扩展名设置 Content-Type

**Cloudflare R2 配置示例**：
1. R2 控制台创建 bucket
2. 创建 API Token（权限：Object Read & Write）
3. 配置自定义域名（Settings → Public Access → Custom Domain）
4. 将域名填入 `S3_PUBLIC_URL`

## 反向代理（推荐）

生产环境建议在前面加 Nginx/Caddy 处理 TLS 和缓存。

### Nginx 配置示例

```nginx
server {
    listen 443 ssl http2;
    server_name your-domain.com;

    ssl_certificate     /etc/ssl/your-domain.com.pem;
    ssl_certificate_key /etc/ssl/your-domain.com-key.pem;

    # 静态资源缓存
    location /_next/static {
        proxy_pass http://127.0.0.1:3000;
        proxy_cache_valid 200 365d;
        add_header Cache-Control "public, immutable";
    }

    # 上传文件缓存（仅 STORAGE_TYPE=local 时需要）
    location /uploads {
        proxy_pass http://127.0.0.1:3000;
        proxy_cache_valid 200 30d;
        add_header Cache-Control "public, max-age=2592000";
    }

    # 应用代理
    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

### Caddy 配置示例（自动 HTTPS）

```
your-domain.com {
    reverse_proxy localhost:3000
}
```

## 数据库管理

### 备份

```bash
# 导出完整备份
docker compose exec db pg_dump -U vela vela > backup-$(date +%Y%m%d).sql

# 定时备份（crontab）
0 2 * * * cd /path/to/project && docker compose exec -T db pg_dump -U vela vela | gzip > /backups/vela-$(date +\%Y\%m\%d).sql.gz
```

### 恢复

```bash
docker compose exec -T db psql -U vela vela < backup.sql
```

### Schema 迁移

```bash
# 开发阶段推送（会自动对比差异）
docker compose exec app npx drizzle-kit push

# 生产阶段迁移（推荐）
docker compose exec app npx drizzle-kit migrate
```

## 环境变量完整参考

| 变量 | 必填 | 默认值 | 说明 |
|------|------|--------|------|
| `DB_PASSWORD` | 是 | `password` | PostgreSQL 密码 |
| `NEXTAUTH_SECRET` | 是 | - | Auth.js 签名密钥（`openssl rand -base64 32`） |
| `SITE_URL` | 是 | `http://localhost:3000` | 站点公开 URL（用于 SEO/OG） |
| `STORAGE_TYPE` | 否 | `local` | 存储方式：`local` 或 `s3` |
| `S3_ENDPOINT` | S3 时 | - | S3 兼容端点 |
| `S3_REGION` | S3 时 | `auto` | S3 区域 |
| `S3_BUCKET` | S3 时 | - | 存储桶名称 |
| `S3_ACCESS_KEY` | S3 时 | - | 访问密钥 |
| `S3_SECRET_KEY` | S3 时 | - | 秘密密钥 |
| `S3_PUBLIC_URL` | S3 时 | - | 文件公开访问域名 |
| `SMTP_ENCRYPTION_KEY` | 否 | - | SMTP 密码加密密钥（`openssl rand -hex 32`） |

## 常见问题

### Q: 容器重启后上传的图片丢失？

确认 docker-compose.yml 中 volume 映射正确：
```yaml
volumes:
  - uploads:/app/public/uploads
```
使用 `docker volume ls` 确认 volume 存在。

### Q: 如何从本地存储迁移到 S3？

1. 将 `public/uploads/` 内容上传到 S3 bucket
2. 修改环境变量 `STORAGE_TYPE=s3` 并配置 S3 参数
3. 数据库中 `media` 表的 `filename` 字段存储的是相对路径（如 `uploads/xxx.webp`），S3 适配器会自动拼接 `S3_PUBLIC_URL`
4. 重启容器

### Q: 如何多实例部署？

1. 使用 `STORAGE_TYPE=s3`（本地存储不支持多实例）
2. 所有实例共享同一个 PostgreSQL
3. 通过负载均衡器（Nginx/ALB）分发流量
4. `NEXTAUTH_SECRET` 必须所有实例一致
